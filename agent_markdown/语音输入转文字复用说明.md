# 语音输入转文字复用说明

## 1. 文档目的

本文记录 HRagent-05 当前“对话预演”页面的语音输入转文字（ASR）实现，供后续维护、迁移和复用。

当前用户流程为：

```text
点击麦克风
  -> 浏览器采集语音
  -> 说话过程中持续更新输入框
  -> 再次点击麦克风停止
  -> 用户检查转写结果
  -> 用户主动点击“发送”
```

识别结果只写入输入框，不会自动作为 HRBP 回复提交。这一约束适用于绩效沟通等敏感场景，复用时应保留。

## 2. 当前架构

```text
RehearsalStep.tsx
  -> useRealtimeAsr.ts
     -> getUserMedia 获取麦克风
     -> Web Audio API 转为 16 kHz、单声道、PCM16
     -> WebSocket /api/v1/asr/realtime
        -> FastAPI asr.py
        -> QwenRealtimeAsrProxy
        -> 上游 Realtime ASR WebSocket
     -> partial/final 事件实时回填输入框

辅助识别路径：
  -> 浏览器 SpeechRecognition 提供本地可见的 interim 文本
  -> 每 750 ms 将累计 PCM 快照提交 /api/v1/asr/transcribe
  -> 停止录音后，在尚无转写文本时提交完整录音
```

安全边界：ASR Key 只保存在后端配置中。浏览器只连接 HRagent 后端，不直接携带上游服务凭证。

## 4. 前端执行过程

### 4.1 启动录音

`useRealtimeAsr.start()` 依次执行：

1. 清空上一轮转写和错误状态。
2. 检查页面是否处于 HTTPS 安全上下文；`localhost` 和回环地址例外。
3. 通过 `navigator.mediaDevices.getUserMedia()` 请求单声道麦克风。
4. 开启降噪、回声消除和自动增益。
5. 启动 `MediaRecorder`，用于停止时的完整录音转写。
6. 浏览器支持时启动 `SpeechRecognition`，并启用 `interimResults`。
7. 创建后端 WebSocket：`/api/v1/asr/realtime`。
8. 启动 Web Audio 音频处理并开始发送 PCM 帧。
9. 最长录音时间到达 60 秒时自动停止当前前端录音。

### 4.2 音频格式

当前实时音频格式为：

| 参数 | 值 |
|---|---|
| 声道 | 单声道 |
| 目标采样率 | 16000 Hz |
| 采样格式 | signed PCM16 |
| 字节序 | little-endian |
| WebSocket 帧 | 二进制 PCM 数据 |

浏览器输入通常是 Float32 和设备原生采样率。`pcm16FromFloat32()` 会完成平均降采样、幅值裁剪和 PCM16 编码。

### 4.3 实时文本显示

识别文本可能来自三个实时来源：

1. 后端 Realtime ASR 的 `partial` 事件。
2. 浏览器 `SpeechRecognition` 的 interim/final 结果。
3. 累计 PCM 的 HTTP 快照转写。

`emitTranscript()` 统一合并这些结果，并调用：

```ts
onPartialTranscript(text, event)
onFinalTranscript(text, event)
```

`RehearsalStep.tsx` 在两个回调中都执行 `setMessage(text)`，因此用户说话时会在输入框看到文字。

### 4.4 停止录音

再次点击麦克风或达到录音上限后：

1. 停止浏览器 SpeechRecognition。
2. 停止音频处理和麦克风轨道。
3. 向 WebSocket 发送 `{"type":"stop"}`。
4. 停止 MediaRecorder。
5. 如果输入框仍没有识别文本，才提交完整录音到 HTTP 转写接口。
6. 已识别文字保留在输入框，等待用户确认和发送。

## 5. 后端接口契约

### 5.1 实时 WebSocket

```text
GET WebSocket /api/v1/asr/realtime
```

浏览器发送：

| 类型 | 内容 |
|---|---|
| Binary | PCM16 音频帧 |
| Text JSON | `{"type":"stop"}` 或 `{"type":"ping"}` |

后端返回的归一化事件：

```json
{"type":"status","message":"ASR connected"}
{"type":"partial","text":"...","preview":"...","emotion":null}
{"type":"final","transcript":"...","emotion":null}
{"type":"error","message":"..."}
```

上游连接建立后，后端发送 `session.update`，配置文本输出、PCM 格式、采样率、语言和 Server VAD。每个浏览器二进制帧会转换为 Base64，并通过 `input_audio_buffer.append` 转发。

### 5.2 HTTP 完整音频转写

```text
POST /api/v1/asr/transcribe
Content-Type: multipart/form-data
```

表单字段：

| 字段 | 必填 | 说明 |
|---|---|---|
| `file` | 是 | WAV、WebM、MP4 或 OGG 音频 |
| `session_id` | 否 | 业务会话 ID |
| `language` | 否 | 默认使用后端 ASR 语言配置 |

响应：

```json
{
  "text": "识别后的文字",
  "audio_emotion": null,
  "duration_seconds": 0.0,
  "provider": "配置的 HTTP ASR 模型"
}
```

## 6. 配置说明

配置位于 `backend/config/.env`，模板位于 `backend/config/.env.example`。真实 Key 不得写入源码、前端、示例文档或日志。

```env
ASR_ENABLED=true
ASR_PROVIDER=qwen
ASR_MODEL=qwen3-asr-flash-realtime
ASR_API_KEY=YOUR_ASR_API_KEY_HERE
ASR_WS_URL=wss://YOUR_REALTIME_ASR_ENDPOINT_HERE
ASR_HTTP_URL=https://YOUR_HTTP_ASR_ENDPOINT_HERE
ASR_HTTP_MODEL=qwen3-asr-flash
ASR_LANGUAGE=zh
ASR_SAMPLE_RATE=16000
ASR_INPUT_AUDIO_FORMAT=pcm
ASR_ENABLE_SERVER_VAD=true
ASR_VAD_THRESHOLD=0.0
ASR_VAD_SILENCE_DURATION_MS=400
ASR_CONNECT_TIMEOUT_SECONDS=15
ASR_SESSION_TIMEOUT_SECONDS=120
ASR_MAX_SESSION_SECONDS=300
```

配置含义：

| 配置 | 用途 |
|---|---|
| `ASR_ENABLED` | 是否启用 ASR |
| `ASR_PROVIDER` | 当前只接受 `qwen` |
| `ASR_MODEL` | Realtime WebSocket 模型 |
| `ASR_API_KEY` | 后端访问上游 ASR 的凭证 |
| `ASR_WS_URL` | 上游实时 WebSocket 地址，不应填写 HTTP 转写地址 |
| `ASR_HTTP_URL` | 非实时完整音频转写地址 |
| `ASR_HTTP_MODEL` | HTTP 转写模型 |
| `ASR_LANGUAGE` | 默认识别语言 |
| `ASR_SAMPLE_RATE` | 上游实时会话采样率，应与前端 16000 Hz 一致 |
| `ASR_INPUT_AUDIO_FORMAT` | 上游实时音频格式，当前为 `pcm` |
| `ASR_ENABLE_SERVER_VAD` | 是否由上游检测说话开始和结束 |
| `ASR_VAD_THRESHOLD` | Server VAD 阈值 |
| `ASR_VAD_SILENCE_DURATION_MS` | 静音达到该时长后结束当前语音段 |
| `ASR_CONNECT_TIMEOUT_SECONDS` | 上游 WebSocket 建连超时 |
| `ASR_SESSION_TIMEOUT_SECONDS` | HTTP 转写请求超时 |
| `ASR_MAX_SESSION_SECONDS` | 后端实时会话最大时长 |

注意：`.env.example` 中还存在 `ASR_MAX_AUDIO_SECONDS` 和 `ASR_MAX_AUDIO_MB`，但当前 `Settings` 和转写路由没有读取或执行这两个限制。复用时不要把它们当作已经生效的服务端校验。

## 7. URL 与反向代理

前端默认 API Base 是 `/api/v1`。`getWsApiBase()` 会根据当前页面协议自动转换：

```text
https://host:8443/api/v1 -> wss://host:8443/api/v1
http://localhost/...     -> ws://localhost/.../api/v1
```

nginx 的 `/api/v1/` 代理必须保留：

```nginx
proxy_http_version 1.1;
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection "upgrade";
proxy_buffering off;
```

远程主机访问麦克风时应使用 HTTPS `8443`。普通 HTTP IP 地址通常不属于浏览器安全上下文，会被禁止使用麦克风。

## 8. 在其他页面复用

前端页面只需复用 Hook，并将 partial/final 都写入用户可见输入框：

```tsx
const [text, setText] = useState('');

const asr = useRealtimeAsr({
  onPartialTranscript: (value) => setText(value),
  onFinalTranscript: (value) => setText(value),
  onError: (message) => showError(message),
});
```

按钮行为：

```tsx
<button onClick={() => (asr.isRecording ? asr.stop() : asr.start())}>
  {asr.isRecording ? '停止录音' : '开始录音'}
</button>
```

复用约束：

1. 页面卸载时由 Hook 自动清理麦克风、计时器、AudioContext 和 WebSocket。
2. partial 和 final 必须更新同一个受控输入框。
3. 不要在 `onFinalTranscript` 中自动提交业务表单。
4. 不要在前端新增 ASR Key。
5. 保持 16 kHz、单声道、PCM16 与后端会话配置一致。

## 9. 安全与隐私

1. ASR Key 仅可存在于后端环境变量或企业密钥管理服务。
2. 当前后端不把原始语音写入本地文件，但浏览器会在录音期间把 PCM 和 MediaRecorder 数据暂存在内存中。
3. 音频会发送到配置的外部或企业 ASR 服务，部署前应确认数据处理、传输区域、保留策略和供应商合规要求。
4. 日志不得打印音频内容、Authorization Header、Cookie 或真实 Key。
5. 当前 `backend/main.py` 将 ASR 路由注册在通用鉴权依赖之外。若复用到更严格环境，应先评估是否需要为 HTTP 和 WebSocket 增加用户身份校验、并发限制及会话配额。

## 10. 当前实现边界

以下内容是维护时需要知道的当前事实：

1. 实际音频处理使用 `ScriptProcessorNode`；`asr-pcm-worklet.js` 已存在，但当前 Hook 没有加载它。
2. HTTP 实时预览每 750 ms 提交一次“截至当前的累计 PCM 快照”，录音越长，请求体越大；稳定的上游 WebSocket partial 事件会关闭该路径。
3. 浏览器 SpeechRecognition 的可用性依赖浏览器及其语音服务，Chrome/Edge 的行为也可能受企业策略或网络限制影响。
4. HTTP 转写路由会把上传音频完整读入内存，当前没有生效的文件大小校验。
5. 前端录音上限是 60 秒，后端 Realtime 会话上限默认是 300 秒，两者不是同一个限制。
6. 多来源文本通过启发式规则合并。若出现重复文字，应检查是否同时收到浏览器预览、HTTP 累计预览和服务端 partial。

这些边界不是新的兜底设计，而是当前代码行为的记录。

## 11. 验证步骤

### 11.1 配置检查

只检查配置是否存在，不输出真实 Key：

```bash
docker compose exec -T backend python -c "from backend.config.settings import get_settings; s=get_settings(); print({'enabled': s.asr_enabled, 'key_set': bool(s.asr_api_key), 'ws_set': bool(s.asr_ws_url), 'http_set': bool(s.asr_http_url), 'model': s.asr_model})"
```

### 11.2 服务检查

```bash
docker compose ps
curl -fsS http://localhost:8111/api/v1/health
curl -k -fsS https://localhost:8443/api/v1/health
```

### 11.3 浏览器验收

1. 使用 Chrome 或 Edge 打开 `https://<host>:8443/app/rehearsal`。
2. 允许该站点访问麦克风。
3. 点击麦克风并连续说一句完整中文。
4. 确认说话过程中输入框持续出现文字。
5. 再次点击麦克风，确认文字保留且没有自动发送。
6. 手动修正文字并点击发送，确认业务流程正常。
7. 打开浏览器 Network，确认存在 `/api/v1/asr/realtime` WebSocket；检查其 Frames 中有二进制音频和 partial/final JSON。

### 11.4 日志检查

```bash
docker compose logs --tail=120 backend
docker compose logs --tail=120 frontend
```

检查日志时不要复制或分享 Authorization、Cookie、Key 等敏感信息。

## 12. 常见问题

| 现象 | 主要检查项 |
|---|---|
| 浏览器无法使用麦克风 | 是否使用 HTTPS/localhost；站点麦克风权限；系统输入设备权限 |
| 一直显示连接中 | nginx Upgrade/Connection 头；后端是否可达；`ASR_WS_URL` 是否为 WebSocket 地址 |
| `HTTP 200` 拒绝 WebSocket | `ASR_WS_URL` 实际是普通 HTTP 接口，需换成实时 WebSocket 地址 |
| 上游返回 401/403 | Key 是否属于对应 ASR 服务；是否开通 Realtime 模型和 WebSocket 权限 |
| 只有停止后才出现文字 | 上游是否返回 partial；WebSocket Frames；浏览器 SpeechRecognition 是否可用 |
| 实时文字重复 | 检查浏览器、HTTP 快照和服务端 partial 是否同时工作；检查文本合并事件来源 |
| 识别延迟逐渐升高 | 检查 WebSocket partial 是否失效，导致累计 HTTP 快照持续执行 |
| 输入框有文字但未发送 | 属于预期行为，必须由用户确认后点击发送 |

## 13. 复用验收标准

复用完成至少满足：

- 麦克风凭证和 ASR Key 不出现在浏览器代码、请求参数和日志中。
- 远程访问使用 HTTPS，WebSocket 使用 WSS。
- 用户说话期间能看到 partial 文本，而不是只能在停止后看到最终结果。
- 停止录音后输入框保留完整文本。
- 转写文本不会自动提交。
- 页面离开后麦克风轨道和 WebSocket 被关闭。
- ASR 失败不会破坏手动文本输入和已有对话流程。
