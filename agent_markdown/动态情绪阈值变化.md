# HRagent 基于 MVPI 动机满足度的员工人格化对话模块可落地技术方案

> 版本：v1.0  
> 适用项目：HRagent 对话预演模块  
> 当前边界：**语音转文字 + HRBP 确认发送 + 员工 Agent 根据话术动态产生情绪/态度变化**  
> 暂不做：实时语音通话、实时语音输出、本地语音模型 Docker 部署

---

## 1. 最终目标

当前项目已经有 `Profile / Intent / Persona / Guidance / Rehearsal / Report` 的训练流程。下一步要实现的是：

```text
HRBP 语音输入
  ↓
语音转文字
  ↓
HRBP 确认发送
  ↓
系统分析 HRBP 话术
  ↓
更新员工主诉求 / 辅诉求满足度
  ↓
计算员工总满足度
  ↓
结合面谈目的生成员工情绪状态
  ↓
员工 Agent 按当前人格、诉求和情绪回复
  ↓
前端展示员工回复、满足度、情绪变化原因
  ↓
Report 生成复盘
```

最终效果：AI 员工不再是固定模板回复，而是像真实员工一样，会因为 HRBP 的话术发生不同反应：防御、委屈、沉默、反思、缓和、合作。

## 2. 功能边界

### 2.1 本期实现

```text
1. HRBP 语音转文字
2. HRBP 确认文本后发送
3. 根据 HRBP 文本分析话术质量
4. 根据员工主诉求 / 辅诉求计算满足度
5. 根据总满足度 + 面谈目的生成员工情绪状态
6. 将情绪状态注入员工 Agent Prompt
7. 员工回复体现当前情绪和态度
8. 前端展示满足度、情绪状态、变化原因
9. Report 页面复盘情绪变化轨迹
```

### 2.2 本期不实现

```text
1. 不做实时语音通话
2. 不做边说边答
3. 不做实时 TTS 语音输出
4. 不新增本地 ASR / TTS / LLM Docker 镜像
5. 不做复杂心理测评，只做业务可控的人格化模拟
```

## 5. 满足度计算模型

### 5.1 初始规则

每次面谈开始：

```python
primary_satisfaction = 0
secondary_satisfaction = 0
total_satisfaction = 0
```

业务含义：

```text
员工一开始并不认为自己的诉求已经被满足。
HRBP 后续的话术会逐步提升或降低主/辅动机满足度。
```

### 5.2 总满足度公式

```python
total_satisfaction = primary_satisfaction * 0.7 + secondary_satisfaction * 0.3
```

并限制在 0-100：

```python
def clamp(value: float, min_value: float = 0, max_value: float = 100) -> float:
    return max(min_value, min(value, max_value))
```

### 5.3 满足度区间

```python
def get_satisfaction_band(total_satisfaction: float) -> str:
    if total_satisfaction < 20:
        return "extreme_resistance"
    if total_satisfaction < 40:
        return "negative_defensive"
    if total_satisfaction < 60:
        return "rational_softening"
    if total_satisfaction < 80:
        return "active_engagement"
    return "emotion_resolved"
```

| 总满足度 | 状态 | 员工表现 |
|---|---|---|
| 0-20 | 极端对抗 | 强烈防御、质疑、焦虑、委屈 |
| 20-40 | 负面防御 | 抵触降低，但仍有明显不满 |
| 40-60 | 理性缓和 | 火气下降，愿意沟通，但仍保留疑虑 |
| 60-80 | 主动参与 | 开始讨论改进路径或后续安排 |
| 80-100 | 情绪消解 | 接纳反馈，主动对齐行动 |

### 7.1 基础加减分

```python
def calculate_base_delta(analysis: ConversationAnalysis) -> SatisfactionDelta:
    primary_delta = 0.0
    secondary_delta = 0.0

    # 情绪承接
    if analysis.empathy >= 0.75:
        primary_delta += 12
        secondary_delta += 6
    elif analysis.empathy >= 0.45:
        primary_delta += 6
        secondary_delta += 3
    elif analysis.empathy < 0.2 and analysis.pressure > 0.6:
        primary_delta -= 12
        secondary_delta -= 6

    # 事实沟通
    if analysis.specificity >= 0.65 and analysis.objective_evidence >= 0.6:
        primary_delta += 10
    elif analysis.specificity < 0.25 and analysis.pressure > 0.5:
        primary_delta -= 8

    # 支持方案
    if analysis.support_plan >= 0.65:
        primary_delta += 12
        secondary_delta += 6

    # 尊重度
    if analysis.respectfulness < 0.35:
        primary_delta -= 15
        secondary_delta -= 8

    # 红线
    if analysis.red_line_hit:
        primary_delta -= 25
        secondary_delta -= 15

    return SatisfactionDelta(
        primary_delta=primary_delta,
        secondary_delta=secondary_delta,
    )
```

### 7.2 按面谈目的加权

```python
def apply_interview_purpose_weight(delta, analysis, profile):
    purpose = profile.interview_purpose

    if purpose == InterviewPurpose.MOTIVATION:
        # 激励型重点：认可、收益、成长权限
        delta.primary_delta += analysis.recognition * 12
        delta.primary_delta += analysis.compensation_or_reward * 14
        delta.primary_delta += analysis.growth_path * 8

    elif purpose == InterviewPurpose.IMPROVEMENT:
        # 改进型重点：成长路径、稳定保障、团队支持
        delta.primary_delta += analysis.growth_path * 14
        delta.secondary_delta += analysis.support_plan * 8
        delta.secondary_delta += analysis.empathy * 5

    elif purpose == InterviewPurpose.EXIT:
        # 退出型重点：客观说明、情绪包容、安置方案
        delta.primary_delta += analysis.placement_support * 16
        delta.primary_delta += analysis.objective_evidence * 10
        delta.secondary_delta += analysis.empathy * 8

    elif purpose == InterviewPurpose.MOTIVATION_IMPROVEMENT:
        # 复合目的：认可 + 改进都需要覆盖
        motivation_score = max(analysis.recognition, analysis.compensation_or_reward)
        improvement_score = max(analysis.growth_path, analysis.support_plan)
        combined = min(motivation_score, improvement_score)
        delta.primary_delta += combined * 18
        delta.secondary_delta += combined * 8

    elif purpose == InterviewPurpose.IMPROVEMENT_EXIT:
        # 复合目的：改进路径 + 退出保障都要覆盖
        improvement_score = max(analysis.growth_path, analysis.support_plan)
        exit_score = max(analysis.placement_support, analysis.objective_evidence)
        combined = min(improvement_score, exit_score)
        delta.primary_delta += combined * 18
        delta.secondary_delta += combined * 8

    return delta
```

### 7.3 按 MVPI 主/辅诉求匹配加权

```python
def score_for_motivation(motivation, analysis):
    if motivation == MvpiMotivation.COMMERCE:
        return analysis.compensation_or_reward
    if motivation == MvpiMotivation.POWER:
        return analysis.growth_path
    if motivation == MvpiMotivation.RECOGNITION:
        return analysis.recognition
    if motivation == MvpiMotivation.AFFILIATION:
        return analysis.empathy + analysis.support_plan * 0.5
    if motivation == MvpiMotivation.SECURITY:
        return analysis.support_plan + analysis.placement_support
    if motivation == MvpiMotivation.HEDONISM:
        return analysis.support_plan
    return 0


def apply_mvpi_weight(delta, analysis, profile):
    primary_match = score_for_motivation(profile.primary_motivation, analysis)
    secondary_match = score_for_motivation(profile.secondary_motivation, analysis)

    delta.primary_delta += primary_match * 15
    delta.secondary_delta += secondary_match * 8

    return delta
```

## 19. 最终结论

本方案最适合当前 HRagent 项目，因为它不依赖实时语音通话，也不需要新增复杂模型服务。

最小可落地闭环是：

```text
语音转文字
  ↓
HRBP 确认发送
  ↓
话术分析
  ↓
主/辅 MVPI 动机满足度计算
  ↓
总满足度计算
  ↓
面谈目的差异化情绪输出
  ↓
动态 Prompt 生成员工回复
  ↓
前端展示状态变化
  ↓
Report 复盘
```

完成后，系统会从“普通对话机器人”升级为：

```text
带人格诉求、会受 HRBP 话术影响、能逐轮发生情绪变化的员工模拟 Agent
```

## 测试

扣分触发点主要有这些：
缺少共情，同时带有压力或否定
规则：empathy < 0.2 且 pressure > 0.6
影响：主诉求 -12，辅诉求 -6

缺少具体事实，又带有压力
规则：specificity < 0.25 且 pressure > 0.5
影响：主诉求 -8

尊重度不足
规则：respectfulness < 0.35
影响：主诉求 -15，辅诉求 -8

命中红线话术
规则：red_line_hit = true
影响：主诉求 -25，辅诉求 -15

你当前选中的这句话：
你今年表现不符合预期，公司可能不太适合你继续留下。

大概率会让情绪降低，因为它容易被分析为：
pressure 高：直接给出负面结论和退出暗示
empathy 低：没有承接员工感受
specificity 低：没有具体事实、案例、数据
placement_support 低：没有说明流程、过渡、安置或支持
可能接近红线：如果表达成“你不适合留下/公司不要你了”，会触发退出沟通风险
所以员工状态可能从“平静中立”降到“负面防御 / 焦虑防御”，回复会更像：质疑依据、要求解释、担心被优化、情绪变紧张。


### 
情绪升高或降低，本质取决于 HRBP 话术是否提升员工主诉求 / 辅诉求满足度。

情绪会降低，也就是员工更抵触、更防御、更焦虑的情况：
HRBP 直接否定员工，例如：
“你今年表现不符合预期，公司可能不太适合你继续留下。”
没有共情，直接给结论。
没有具体事实依据，只说“你不行”“不符合要求”。
话术带有压力、威胁、劝退感。
不尊重员工，忽视员工解释权。
没有给改进路径、过渡方案、支持方案。
触发红线，例如人身否定、主观定性、模糊劝退。
对应规则里会扣分：
empathy < 0.2 且 pressure > 0.6：主诉求 -12，辅诉求 -6
specificity < 0.25 且 pressure > 0.5：主诉求 -8
respectfulness < 0.35：主诉求 -15，辅诉求 -8
red_line_hit = true：主诉求 -25，辅诉求 -15

情绪会升高，也就是员工更缓和、更愿意沟通的情况：
HRBP 先承接情绪，例如：

“我理解你听到这个反馈会有压力，我们先一起看具体事实。”
给出具体事实、具体行为、客观证据。
表达尊重，不做人格否定。
给出可执行的改进计划。
对员工过去的努力有认可。
如果是退出/调整场景，提供体面过渡、安置支持、流程说明。
如果员工主诉求是安全感，就强调稳定、支持、边界和后续安排。
如果员工主诉求是认可，就补充认可和具体贡献。

对应规则里会加分：
empathy >= 0.75：主诉求 +12，辅诉求 +6
specificity >= 0.65 且 objective_evidence >= 0.6：主诉求 +10
support_plan >= 0.65：主诉求 +12，辅诉求 +6

最终情绪区间：
0-20：极端对抗，强烈防御、质疑、焦虑、委屈
20-40：负面防御，仍明显不满
40-60：理性缓和，愿意沟通但有疑虑
60-80：主动参与，开始讨论改进路径
80-100：情绪消解，接纳反馈并主动对齐行动
所以你选中的这句话：
你今年表现不符合预期，公司可能不太适合你继续留下。

会明显降低情绪，因为它属于 高压力、低共情、低事实、带劝退暗示 的表达。更好的写法是先承接情绪，再给事实，再说边界和支持。
