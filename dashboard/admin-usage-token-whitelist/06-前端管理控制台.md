# 06：前端管理控制台

保留路由 `/admin`，将 `frontend/src/pages/AdminPage.tsx` 重构为 Overview、Users & Tokens、Usage Records、Whitelist & Accounts、Security Audit 五个 Tab。新增 `frontend/src/api/admin.ts` 与 `frontend/src/types/admin.ts`，并按需要拆分页面和组件到 `pages/admin/`、`components/admin/`。

## 信息架构

Overview 展示 KPI、Token 趋势、Top 用户、模型/任务分布和最近错误；Users 提供筛选、用量表和详情 Drawer；Usage Records 提供日期/模型/任务/状态筛选与 CSV；Whitelist 管理账号与授权；Audit 提供登录、管理员操作、API 错误三个子 Tab。

顶部显示 Bosch 标识、`HRagent Admin Console`、当前管理员、统计范围、刷新时间、刷新与返回工作台。KPI 至少包括活跃用户、总 Token、LLM 调用、成功率、平均耗时、白名单、业务 Session、API 错误；无后端同期数据不得伪造环比。

图表优先 React + 原生 SVG，不强制 ECharts/Recharts：Token 趋势、Top 用户水平柱状图、模型环形分布、任务分布。需响应式、Hover tooltip、空数据、单位、估算标记及非纯颜色状态表达；保持 Bosch 配色、CSS 变量、克制企业风格，不用炫目渐变/玻璃拟态。

表格：用户表显示状态、白名单、登录/活动、业务 Session、API、LLM、输入/输出/总 Token、错误和操作；调用表显示时间、用户、任务、模型、流式、Token 来源、耗时和状态，估算值显示 `≈`。白名单页使用 Modal/Drawer、搜索/分页/状态筛选、危险操作二次确认和根管理员锁定。审计页禁止显示任何敏感信息。

## 权限与请求行为

前端入口除了 `role === 'admin'`，还应核对邮箱是根管理员；但最终权限只依赖后端。Admin API 使用 `credentials: 'include'`、统一错误解析、AbortController；筛选变化取消旧请求；401 跳登录，403 跳首页并提示；CSV 使用 Blob 下载；浏览器内不缓存密码。移动端表格允许横向滚动。
