# 01：范围与安全模型

## 建设目标与边界

在现有 `/admin` 上增加用户使用情况、Token 可视化、白名单/账号管理和安全审计。根管理员为 `aah5sgh@bosch.com`。可查看用户、登录/活动时间、业务会话、API/LLM 调用、输入/输出/总 Token、模型/任务分布、耗时与数据来源；支持筛选、分页、CSV 导出。

本期不实现 Bosch OIDC/SSO、真实费用、多级 RBAC、用户自助 Token、硬性限额/限流、Grafana 业务管理或 Prompt/完整回复存储。

## 架构

```text
React /admin → FastAPI /api/v1/admin/* → PostgreSQL
                    ├─ AdminAnalytics / AdminAccount / AdminAudit
                    ├─ Request Context + API request audit
                    └─ ModelFarmLangChainChatModel → UsageTrackingService
```

PostgreSQL 包含既有 `app_users`、`auth_whitelist`、`auth_audit_log`、`sessions`，以及新增 `llm_usage_events`、`api_request_events`、`admin_action_audit_log`。Redis 继续用于登录 Session 和活跃 Session 统计。

## 配置与超级管理员依赖

在 `backend/config/settings.py` 增加：

```python
admin_super_email: str = "aah5sgh@bosch.com"
admin_console_enabled: bool = True
admin_usage_tracking_enabled: bool = True
admin_usage_estimation_enabled: bool = True
admin_usage_retention_days: int = 365
admin_api_audit_retention_days: int = 90
admin_export_max_rows: int = 50000
admin_default_range_days: int = 7
```

环境变量对应 `ADMIN_SUPER_EMAIL`、`ADMIN_CONSOLE_ENABLED`、`ADMIN_USAGE_TRACKING_ENABLED`、`ADMIN_USAGE_ESTIMATION_ENABLED`、`ADMIN_USAGE_RETENTION_DAYS`、`ADMIN_API_AUDIT_RETENTION_DAYS`、`ADMIN_EXPORT_MAX_ROWS`、`ADMIN_DEFAULT_RANGE_DAYS`。

在 `backend/core/auth_dependency.py` 新增 `require_super_admin_session`：当前会话必须同时满足 `role == "admin"` 且规范化后的邮箱等于 `settings.admin_super_email`，否则返回 403。所有 `/api/v1/admin/*` 必须使用该依赖，不能只使用普通管理员依赖。

## 生产与不可变保护

当 `ENVIRONMENT != local`，必须同时要求 `AUTH_ENABLED=true`、`AUTH_COOKIE_SECURE=true`、`ADMIN_CONSOLE_ENABLED=true` 与正确的 `ADMIN_SUPER_EMAIL`。当认证关闭时，`/admin` 不可进入，所有管理 API 返回 403，不能使用模拟管理员绕过。

后端必须强制保护根管理员：不得删除、停用、移出白名单、降级为 user、强制清除全部 Session（显式退出除外），且其他账号不可重置其密码。前端禁用按钮只是辅助。
