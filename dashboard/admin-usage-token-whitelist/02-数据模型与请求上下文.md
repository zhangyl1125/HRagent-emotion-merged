# 02：数据模型与请求上下文

## 迁移与初始化

新增 `backend/db/migrations/0003_add_admin_usage_tables.sql`，并将同一份幂等建表逻辑同步加入 `backend/repositories/postgres_repository.py::init_schema()`；项目运行时会调用后者，不能只添加迁移文件。

## 新增事实与审计表

`llm_usage_events`：以 `call_id UUID UNIQUE` 为单次调用去重键，记录 `trace_id`、可空 `user_id`、`email_snapshot`、可空业务 session、任务、provider、model、流式标记、输入/输出/reasoning/cached/total token、来源、状态、HTTP 状态、耗时、重试次数、Provider request ID、错误码及仅含数字的 usage metadata。约束：来源仅 `provider|estimated|unavailable`，状态仅 `success|error|cancelled`，所有非空 Token 非负。为 created/user/email/model/task/status/session 建立时间相关索引。

`api_request_events`：记录 `trace_id`、可空用户/邮箱快照、HTTP method、路由模板、状态码、耗时和 UTC 创建时间；为 created、user、route、status 建索引。禁止保存 query、body、response、Cookie、Authorization、用户输入和上传文件。

`admin_action_audit_log`：记录 trace、操作者、动作、目标、脱敏 before/after、是否成功、原因和时间；不得保存密码或密码哈希。为 created、操作者、目标建索引。

扩展 `auth_whitelist`：增加 `created_by`、`updated_by`、`updated_at`，并令白名单更新同时写入 note 与这些字段。初期不建立汇总表；仅在用量超过约 100 万行且查询明显变慢时考虑。

## 请求关联

扩展 `backend/core/session_context.py`，新增不可变 `UsageRequestContext(trace_id, user_id, email, role, business_session_id)` 的 `ContextVar`，但保留 `get_current_auth_user_id()` 的兼容行为。

新增 `backend/middleware/request_context.py`：每个 HTTP 请求生成 UUID trace ID；可复用合法且经长度/字符清洗的 `X-Request-ID`；将其写入 ContextVar、`request.state.trace_id` 和响应头；请求结束必须 reset，防止并发串上下文。

在 `get_current_session()` 认证成功后填充 `request.state.auth_user_id/email/role` 并更新 usage context。具有业务 `session_id` 的入口用 `bind_business_session(session_id)` 临时绑定，优先 guidance、rehearsal、report、employee simulation、profile extraction、intent recognition；缺少业务 session 可为空，但 user 和 trace 不得丢失。
