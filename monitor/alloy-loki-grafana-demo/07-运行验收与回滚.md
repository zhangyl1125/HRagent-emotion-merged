# 07：运行、验收与回滚

## 启动与日常操作

启动业务：

```bash
docker compose up -d
```

启动可观测性栈：

```bash
docker compose --env-file observability/.env -f observability/docker-compose.yml up -d
```

查看状态、资源和日志：

```bash
docker compose --env-file observability/.env -f observability/docker-compose.yml ps
docker stats --no-stream hragent-alloy hragent-loki hragent-grafana
docker compose --env-file observability/.env -f observability/docker-compose.yml logs --tail=200 alloy loki grafana
```

停止可观测性服务（默认保留数据卷）：

```bash
docker compose --env-file observability/.env -f observability/docker-compose.yml down
```

除非用户明确确认不再需要历史日志和 Dashboard 数据，否则禁止 `down -v`。

## 验收清单

- [ ] 原有业务功能和 API schema 无回归。
- [ ] 根 Compose 校验通过，四个业务服务都启用 Docker 日志轮转。
- [ ] 可观测性 Compose、Alloy `validate`、Alloy `fmt --test` 和 Dashboard JSON 校验通过。
- [ ] Alloy、Loki、Grafana 都运行且健康；Alloy 仅采集 `hragent-05`。
- [ ] Loki 数据源和 Dashboard 在全新 Grafana 卷自动 provision；五个面板可执行。
- [ ] backend、frontend、postgres、redis 四类日志均可检索。
- [ ] Loki 为 7 天保留；Loki/Alloy 无主机端口；Grafana 仅绑定 `127.0.0.1`。
- [ ] Grafana 禁止匿名访问和公开注册。
- [ ] 代码和日志无明文密码、Token、API Key、HR 敏感正文或原始 session ID。
- [ ] 监控容器没有 OOM 或重启循环；记录了资源和 Loki 卷占用。
- [ ] 已提供变更列表、真实验证输出摘要和回滚命令。

## 回滚

先停止监控服务，业务 Compose 不应受到影响：

```bash
docker compose --env-file observability/.env -f observability/docker-compose.yml down
```

若业务 Docker 日志轮转有兼容性问题，只删除根 `docker-compose.yml` 中新增的 `x-default-logging` 与各服务 `logging: *default-logging`，随后：

```bash
docker compose -f docker-compose.yml config
docker compose up -d
```

默认保留 `hragent_alloy_data`、`hragent_loki_data`、`hragent_grafana_data` 三个卷。只有明确确认无需历史数据时才能删除。

## 必须停止并报告的情况

不得猜测或绕过以下问题：

- Compose project name 或服务名不符合约定；
- 固定镜像不存在；
- Docker Compose 不支持资源限制字段；
- Alloy 校验失败、Loki 解析失败或 Dashboard 无法 provision；
- 主机资源不足并出现内存/磁盘风险；
- 日志出现大量 HR 敏感数据；
- Docker Socket 无法挂载。

报告应包含准确错误、影响和最小修复方案。
