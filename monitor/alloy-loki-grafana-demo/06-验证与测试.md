# 06：验证与测试

## 验证脚本

创建 `scripts/verify_observability.sh`。要求：

1. 使用 `set -Eeuo pipefail`；
2. 检查 `docker` 和 `docker compose`；
3. 检查 `observability/.env` 存在；
4. 校验业务与可观测性 Compose；
5. 检查 Loki `/ready`、Grafana `/api/health`；
6. 检查 `hragent-alloy`、`hragent-loki`、`hragent-grafana` 是 running/healthy；
7. 输出 Loki label 查询是否包含 `service`，并查询最近 5 分钟 HRagent 日志数量；
8. 任意失败返回非零；不得打印 Grafana 密码。

参考骨架：

```bash
#!/usr/bin/env bash
set -Eeuo pipefail
ROOT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
OBS_COMPOSE="$ROOT_DIR/observability/docker-compose.yml"
OBS_ENV="$ROOT_DIR/observability/.env"
command -v docker >/dev/null 2>&1 || { echo "ERROR: docker is not installed" >&2; exit 1; }
docker compose version >/dev/null
[[ -f "$OBS_ENV" ]] || { echo "ERROR: $OBS_ENV does not exist" >&2; exit 1; }
docker compose -f "$ROOT_DIR/docker-compose.yml" config >/dev/null
docker compose --env-file "$OBS_ENV" -f "$OBS_COMPOSE" config >/dev/null
for container in hragent-alloy hragent-loki hragent-grafana; do
  state="$(docker inspect -f '{{.State.Status}}' "$container" 2>/dev/null || true)"
  [[ "$state" == "running" ]] || { echo "ERROR: $container is not running" >&2; exit 1; }
done
curl -fsS http://127.0.0.1:3000/api/health >/dev/null
docker exec hragent-loki wget -qO- http://127.0.0.1:3100/ready >/dev/null
end_ns="$(date +%s%N)"; start_ns="$((end_ns - 300000000000))"
query_result="$(docker exec hragent-loki wget -qO- "http://127.0.0.1:3100/loki/api/v1/query_range?query=%7Bcompose_project%3D%22hragent-05%22%7D&start=${start_ns}&end=${end_ns}&limit=20" || true)"
[[ "$query_result" == *'"status":"success"'* ]] || { echo "ERROR: Loki query did not succeed" >&2; exit 1; }
echo "OK: observability stack is healthy"
```

不得假定镜像一定包含 `wget`、`curl`、`jq`。根据实际镜像内工具改写容器检查；宿主机可用 `curl`。

## 静态检查

全部必须成功：

```bash
docker compose -f docker-compose.yml config
docker compose --env-file observability/.env -f observability/docker-compose.yml config
python -m json.tool observability/grafana/dashboards/hragent-logs.json >/dev/null
```

另须执行 04 文档中的 Alloy `validate` 和 `fmt --test`。

## 运行测试

1. 启动当前 HRagent 和可观测性栈。
2. 调用健康检查并访问至少一个后端 API；刷新前端以产生 Nginx 日志。
3. 60 秒内在 Grafana Explore 查询以下日志：

```logql
{compose_project="hragent-05"}
{compose_project="hragent-05", service="backend"}
{compose_project="hragent-05", service="frontend"}
{compose_project="hragent-05", service="postgres"}
{compose_project="hragent-05", service="redis"}
```

## 隐私、资源与保留检查

抽查最近 200 条后端日志，确认无 Authorization、Cookie、API Key、密码、员工完整邮箱、对话/文档正文、完整文件名或原始 session ID。自动扫描只能辅助，不能代替人工抽查。

稳定运行 10 分钟后检查：

```bash
docker stats --no-stream hragent-alloy hragent-loki hragent-grafana
docker system df -v
docker volume inspect hragent_loki_data
du -sh <loki-volume-mountpoint>
```

不得有重启循环、OOMKilled、持续内存逼近限额、Loki 写入失败，或 Alloy 持续 429/连接失败/丢弃日志。立即确认 Loki 加载 `retention_period: 168h`、Compactor 已启动、索引周期 24h；真正删除效果须在运行超过 7 天后复验。
