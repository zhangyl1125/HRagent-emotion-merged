# 阶段 06：50 并发验收

## 目标

在隔离环境验证 50 个活跃完整流程的 Token、稳定性、队列和质量目标，形成是否允许发布的最终结论。

## 现状

- 完整流程包含登录、建会话、资料、Guidance、15 轮预演、结束预演和 CoachReport。
- 固定账号模式要求每个虚拟用户拥有已启用的专用白名单账号。
- 外部模型 API 压测可能消耗额度，必须先获得测试窗口和配额授权。

## 改动

1. 在隔离数据库或隔离测试环境生成 50 个专用账号，并加入白名单与 Locust 凭据表。
2. 密码只从环境变量注入；脚本只保存无秘密的账号标识和随机 nonce。
3. 按 1、10、25、50 用户逐级执行，每级未通过不得继续升压。
4. 50 用户以每秒 1 用户爬坡，每用户只运行一次完整 15 轮流程，最长测试窗口 60 分钟。
5. 结束后停用测试账号，不删除业务表、历史用量或其他用户数据。

## 接口与配置

- 使用现有 Locust 环境变量和固定账号表；不得把密码写入脚本、文档或命令历史。
- 质量门禁新增完整流程完成数、Guidance/Report 完成数、队列 P95 和每流程 Token。
- 不修改生产白名单，不关闭认证进行验收。

## 测试

- 1 用户：完整业务和质量冒烟。
- 10 用户：验证跨 worker 队列、singleflight 和重试。
- 25 用户：验证外部 API 限流与恢复。
- 50 用户：验证完整目标并保存 CSV、JSON、HTML 和脱敏数据库汇总。
- 每级同时检查后端、PostgreSQL、Redis、Grafana 和外部 API 错误分布。

## 验收

- 50 个用户中至少 49 条完整流程在 60 分钟内结束。
- 请求总失败率低于 1%，Guidance/Report 不完整数为 0，未处理 429/5xx 为 0。
- 模型队列等待 P95 不超过 120 秒且没有租约泄漏。
- 每条完成流程平均输入 Token 不超过阶段 01 基线的 50%。
- 黄金质量门禁全部通过，服务在测试后恢复稳定状态。

## 回滚

立即停止 Locust，停用本次测试账号，关闭优化功能开关并恢复上一阶段配置；不执行 `down -v`、不清空 Redis/PostgreSQL。

## 停止条件

未获得外部 API 压测授权、账号少于 50、监控不可用、前一级失败、出现真实员工数据或共享环境受影响时立即停止。
