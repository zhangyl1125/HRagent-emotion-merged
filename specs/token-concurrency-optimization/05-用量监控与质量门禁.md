# 阶段 05：用量监控与质量门禁

## 目标

让 Token 降幅、队列行为、缓存效果和输出质量可被持续验证，并在回归时阻止进入 50 并发阶段。

## 现状

- `llm_usage_events` 已提供 Token、模型、任务、重试、耗时和状态。
- Grafana 已展示部分模型用量，但缺少每完整流程 Token、队列等待、singleflight 和缓存收益。
- 现有性能门禁不能单独证明业务流程完整或输出质量稳定。

## 改动

1. 使用现有 `usage_metadata` 保存非敏感元数据：Prompt 版本、上下文 turn 数、记忆字符数、队列等待、缓存命中、singleflight 角色和是否遇到限流。
2. 增加每任务及每完整流程的输入/输出 Token、调用数、重试数和耗时查询。
3. Grafana 增加 Token/流程、任务占比、队列等待、容量错误、缓存命中和 singleflight 节省调用面板。
4. 建立 10 个合成黄金用例，覆盖不同 Persona、难度、意图、事实冲突和红线场景。
5. CI 同时执行 Schema、证据引用、禁止项、分数漂移、Token 预算和 0 请求门禁。

## 接口与配置

- 不新增数据库列或业务 API。
- `usage_metadata` 禁止写入 Prompt、消息正文、邮箱、Session ID、Cookie 或 Provider 请求正文。
- Grafana 查询只能读取用量聚合和脱敏技术字段。

## 测试

- 单测 metadata 字段白名单和敏感值拦截。
- 对同一合成完整流程核对数据库、Grafana SQL 与 Locust 汇总一致。
- 运行 10 个黄金用例，验证 Schema、证据原文存在性、禁止承诺和评分漂移。
- 人为制造 Token 超预算、0 请求和容量错误，确认门禁失败。

## 验收

- 每条完整流程可计算输入 Token、输出 Token、调用数、队列等待和缓存收益。
- 阶段 02 的输入 Token 降幅持续不低于 50%。
- 黄金用例 Schema 通过率 100%，禁止项为 0，Coach 总分漂移不超过 10 分。
- Dashboard 和日志均不包含敏感正文或真实凭据。

## 回滚

回滚新增面板和 metadata 写入字段；保留已有用量事件，不修改或删除历史数据。

## 停止条件

指标只能通过记录 Prompt/回复正文实现、Grafana 需要高权限业务账号，或必须新增数据库迁移时停止并重新确认。
