# HRagent Token 降耗与外部 API 高并发 Specs

本目录是 `hragent-token-concurrency` Skill 的阶段事实源。所有代码实施必须从阶段 01 开始，只能在当前阶段验收通过后进入下一阶段。创建这些 Specs 不等于授权执行其代码、数据或运行环境变更。

## 总目标

- 仅使用现有外部模型 API，不使用、不合并、不移植 Ollama 能力。
- 相同合成资料和 15 轮对话下，每条完整流程输入 Token 至少降低 50%。
- 50 个活跃完整流程中至少 49 条完整结束，请求总失败率低于 1%。
- Guidance、15 轮预演和 CoachReport 均收到完整结束事件。
- 保持现有成功响应、SSE 事件名称、业务 Schema、证据约束和六步用户流程。

## 全局实施规则

- 每阶段开始前确认分支、工作区、服务状态和上一阶段证据。
- 测试数据必须为合成数据；密码、API Key 和连接串只通过环境变量注入。
- 不保存 Prompt、模型完整回复、员工正文、Cookie、Session Token 或真实业务会话标识。
- 不缓存完整 AI 员工回复；Redis 异常不得放开为无限并发。
- 任何数据库迁移、鉴权、模型、服务地址、成功响应或 Docker 变化均需重新授权。
- 不自动提交、推送、合并、删除数据或清理 Volume。
