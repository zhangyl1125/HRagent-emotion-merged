# 优化模式快速决策

根据已测量热点选择模式，不根据理论收益或旧案例直接实施。

| 热点信号 | 首选模式 | 主要风险 | 必须验证 |
| --- | --- | --- | --- |
| 静态规则或字段重复 | Prompt 去重 | 误删安全约束 | 规则覆盖与黄金样本 |
| Few-shot 冗长 | 缩短或减少示例 | 边界输出漂移 | 分层样本质量 |
| 输出长尾明显 | 任务级输出预算 | 截断 | 历史 P95 与完成标记 |
| 多次调用携带共同上下文 | 调用合并或批处理 | 单次失败范围扩大 | 映射和部分失败 |
| 长对话持续增长 | 滑动窗口与滚动记忆 | 早期事实丢失 | 关键事实保留 |
| 工具或资料大量闲置 | 按需加载 | 漏载必要能力 | 任务覆盖率 |
| 检索片段过多或重复 | RAG 去重与预算 | 引用缺失 | 召回和证据覆盖 |
| 大块稳定前缀重复 | Prompt Cache | 动态字段破坏命中 | Provider 缓存指标 |
| 相同确定性输入重复 | 结果缓存 | 失效和隔离错误 | TTL、版本和权限 |
| 并发相同未命中 | Singleflight | 锁停滞 | 租约和所有者恢复 |
| 重试或重连重复计费 | 幂等与有限重试 | 错误分类 | 重试放大率 |
| 任务复杂度差异明显 | 模型路由或级联 | 质量漂移和升级放大 | 黄金样本与回退 |

## 推荐顺序

1. 先修复计量和非零质量门禁。
2. 再处理重复 Prompt、重复字段和明显失控的输出。
3. 然后压缩历史、工具结果和检索上下文。
4. 再处理缓存、Singleflight、幂等和重试放大。
5. 最后在单独授权与质量证据下考虑模型路由、微调或部署变化。

## 停止判断

出现以下任一情况时停止实施：

- 无法获得可比的优化前后基线。
- 需要读取或记录真实凭据、Prompt、完整回复或个人数据。
- 改动超出当前授权范围。
- 质量、Schema、安全、证据或流式契约回归。
- 节省仅来自估算，Provider 数据或账单无法验证。
