## 1. 计量口径

按完整、成功的用户可见业务结果计量。单次调用变短不等于完整流程更便宜；若引发重试、修复调用或缓存失效，总费用可能上升。

至少记录：

| 字段 | 用途 |
| --- | --- |
| 流程、任务、功能 | 将用量归因到业务结果 |
| Provider、模型、版本 | 关联能力、价格和行为变化 |
| 输入、输出、推理、缓存 Token | 区分费用来源 |
| 用量来源 | 区分 Provider 返回、估算和不可用 |
| 其他计费单位 | 记录 Embedding、Rerank、语音、图像或工具费用 |
| 调用、重试、状态 | 识别放大和失败成本 |
| 排队、首字节、总耗时 | 区分吞吐与模型耗时 |
| 价格、币种、生效日期、来源 | 复现历史费用 |

通用费用模型：

```text
总费用 = 输入 Token / 计价单位 × 输入单价
       + 缓存输入 Token / 计价单位 × 缓存输入单价
       + 输出 Token / 计价单位 × 输出单价
       + 推理 Token / 计价单位 × 推理单价
       + 请求费、工具费及其他 Provider 计费项
```

语音按时长、Rerank 按文本或文档数、图像按尺寸或张数计费时，使用独立字段，不要伪装成 LLM Token。

## 2. 热点审计

沿入口到 Provider 还原完整调用图，逐项检查：

- 相同系统规则是否在多条消息或多次子调用中重复。
- Few-shot 是否多于证明输出稳定所需的最小数量。
- 是否每轮都序列化完整历史、用户档案或大型工具 Schema。
- 工具结果是否未经筛选、截断或摘要直接回灌模型。
- RAG 是否重复查询、重复片段、Top-K 过大或 Rerank 范围失控。
- 输出是否明显低于配置上限，或因截断触发修复调用。
- 并发未命中、断线重连或重试是否重复执行同一工作。
- 缓存键是否因时间戳、字段顺序或动态前缀导致低命中。

优先级评分可使用：

```text
优先级 = 可验证节省 × 调用频率 × 回滚容易度
       ÷（质量风险 × 实施成本）
```

评分只用于排序，不替代真实基线。

## 3. 三层优化模式

### 3.1 Prompt 层

| 模式 | 适用信号 | 风险 | 验证 |
| --- | --- | --- | --- |
| 规则去重 | 同义约束重复 | 误删安全或格式要求 | 规则覆盖和黄金样本 |
| Few-shot 压缩 | 示例冗长或重复 | 边界输出漂移 | 分层样本解析率 |
| 结构化输出 | 分类、提取、多栏目结果 | Schema 修复调用 | 首次解析成功率 |
| 输出预算 | 输出长尾明显 | 截断 | 历史 P95、完成标记 |
| 稳定前缀 | 大块静态上下文重复 | 动态字段破坏命中 | Provider 缓存指标 |

不要假设某种语言必然更省 Token；用目标模型的 Tokenizer 和真实任务样本比较。不要要求模型输出冗长思维过程，只要求必要结论、证据和可审计字段。

### 3.2 Context 层

- 只加载当前任务需要的工具定义、参考资料和用户字段。
- 为工具描述、工具结果、记忆、历史和检索证据分别设置预算。
- 先用确定性规则筛选，再考虑用模型摘要；摘要调用本身必须计费。
- 对重复内容按稳定标识或内容摘要去重。
- 在删除上下文前定义必须保留的事实、证据和安全字段。

### 3.3 Harness 层

| 模式 | 何时真正省 Token | 主要风险 |
| --- | --- | --- |
| 调用合并 | 子调用重复携带共同上下文 | 单次失败范围扩大 |
| 批处理 | 多任务共享指令或上下文 | 映射错误、上下文超限 |
| 确定性缓存 | 相同输入产生稳定结果 | 失效和隔离错误 |
| 语义缓存 | 相似问题可安全复用答案 | 错答、过期、串租户 |
| Singleflight | 并发执行相同缓存未命中 | 锁停滞、所有者失效 |
| 幂等与有限重试 | 避免重连或超时后的重复计费 | 错误分类导致漏重试 |
| 模型路由或级联 | 大量任务可由低成本路径达标 | 升级放大、质量漂移 |

## 4. 对话上下文

组合使用滑动窗口和结构化滚动记忆：

```text
稳定规则
+ 记忆 {事实、数字、承诺、分歧、纠正、待确认项}
+ 最近 N 个完整轮次
+ 当前输入（仅一次）
+ 有界检索证据
```

规则：

- 明确定义一个轮次是单条消息还是用户/助手消息组，并保持一致。
- 只将完整轮次移出窗口，将其中受保护信息增量合并进记忆。
- 用字符数或估算 Token 限制记忆；达到上限时压缩字段，不静默删除受保护事实。
- 排除隐藏推理、推断性诊断、重复描述和完整对话副本。
- 当前输入不得同时出现在历史、记忆或检索内容中。
- 摘要器失败时回退到安全的原路径，不得生成未经验证的事实。

边界测试：0、1、N、N+1 个轮次；超长消息；多语言；数字、日期、否定、承诺和纠正；旧会话无记忆字段；重复压缩不无限增长。

## 5. 检索与缓存

### 5.1 RAG 预算

- 记录查询次数、Embedding 输入、候选数、Rerank 数、最终片段数和最终上下文 Token。
- 先做权限和元数据过滤，再执行高成本 Rerank。
- 分别限制候选、Rerank、最终 Top-K、单片段和总上下文。
- 保留引用所需的来源标识和最少原文，不能为了省 Token 破坏证据链。

### 5.2 Prompt Cache

- 将稳定的系统规则、工具 Schema 和公共资料置于前缀，动态内容置后。
- 规范化字段顺序和序列化方式，避免非语义变化破坏前缀。
- 通过 Provider 返回的缓存写入、读取或命中 Token 验证，不根据“前缀看起来相同”推断。
- 同时报告逻辑输入 Token、缓存 Token、实际费用和延迟。

### 5.3 应用结果缓存

缓存键应包含所有影响语义的输入：

- 租户、授权范围，以及必要时的用户和会话。
- 任务、Provider、模型、参数、Prompt 版本和输出 Schema 版本。
- 知识库或索引版本、规范化检索输入和稳定排序的业务输入。

排除非语义时间戳；对敏感字段使用摘要，不把原文放进可观察的键名。定义 TTL、失效、容量、降级和隔离测试。

## 6. 模型选择与部署

仅在任务级黄金样本和明确授权下考虑：

- **路由**：根据任务类型、复杂度、上下文长度和能力要求选择路径。
- **级联**：低成本路径通过确定性或独立质量门禁后返回，否则升级；将升级调用计入总费用。
- **微调**：适合高频、稳定任务；将训练、数据治理、评测和版本维护计入回收期。
- **本地或混合部署**：比较硬件、能耗、容量、运维、故障、延迟和质量后的总拥有成本。

不得根据旧价格表、模型名称或理论能力直接切换。价格配置放在业务逻辑之外，并记录币种、生效日期和来源。

## 7. 并发、批处理与重试

- 按 Provider 和模型设置有界并发与有界等待队列。
- 多 Worker 共享配额时，使用 Redis 或等效跨 Worker 协调。
- 协调器不可用时采用保守本地上限，不能退化为无限调用。
- 区分限流、超时、临时服务端错误和永久请求错误。
- 使用带抖动的指数退避，尊重 `Retry-After`，限制最大尝试次数和总时限。
- 为重连、任务和批处理使用稳定幂等键。
- Singleflight 租约应有所有者令牌、安全过期和所有者失效恢复。
- 并行独立调用只降低墙钟时间；合并调用必须验证映射、部分失败和上下文窗口。

## 8. 质量门禁与报告

实施前定义：

- 最少请求数、最少完整流程数和最大失败率。
- Schema 首次解析成功率及流式事件名称、顺序和完成标记。
- 关键事实、引用、安全规则和个性化保留率。
- 相对黄金样本允许的质量漂移。
- 最大重试放大、限流率、回退率和队列等待。
- 完整业务结果的 Token 与费用目标。
