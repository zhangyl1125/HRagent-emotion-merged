# Prompt 与上下文优化

## Prompt 审计

逐项检查：

- 删除同义重复规则和可由 Schema 表达的说明。
- 删除未被任务使用的字段和固定背景。
- 将 Few-shot 缩减为覆盖核心格式和边界的最小集合。
- 对分类和提取任务使用标签或结构化输出，减少自然语言包装。
- 根据历史输出分布设置任务级上限，不使用统一的大上限。
- 不要求模型输出冗长思维过程，只要求结论、证据和可审计字段。

每次删除前标记其职责：安全、格式、业务规则、风格或示例。没有对应测试的规则不得盲目删除。

不要假设某种语言必然更省 Token；使用目标模型 Tokenizer 和真实任务样本比较。

## 稳定前缀

按以下顺序组织有利于 Prompt Cache 的输入：

```text
稳定系统规则
+ 稳定工具 Schema
+ 稳定公共资料
+ 会话级上下文
+ 当前动态输入
```

规范字段顺序和序列化格式，将时间戳、请求 ID 等非语义动态值移出稳定前缀。通过 Provider 返回的缓存指标验证命中。

## 工具与资料按需加载

- 只加载当前任务可能调用的工具定义。
- 精简工具名称、描述、参数说明和示例。
- 为工具结果设置字段白名单、条数和字符或 Token 上限。
- 先用确定性规则筛选，再考虑模型摘要；摘要调用必须计费。
- 将必要上下文和可选上下文分层，缺少可选内容时允许降级。
- 测试每类任务仍能发现并调用必要工具。

## 滑动窗口与滚动记忆

按以下顺序组装多轮上下文：

```text
稳定规则
+ 结构化记忆 {事实、数字、承诺、分歧、纠正、待确认项}
+ 最近 N 个完整轮次
+ 当前输入（仅一次）
+ 有界检索证据
```

规则：

- 明确定义一个轮次是单条消息还是用户/助手消息组，并保持一致。
- 只将完整轮次移出窗口。
- 将移出轮次中的受保护信息增量合并进记忆。
- 用字符数或估算 Token 限制记忆；达到上限时压缩字段。
- 不静默删除事实、标识符、数字、否定、承诺、冲突和待确认项。
- 排除隐藏推理、推断性诊断、重复描述和完整对话副本。
- 当前输入不得同时存在于历史、记忆或检索内容。
- 摘要器失败时回退到旧路径，不生成未经验证的事实。

## 上下文边界测试

覆盖：

- 0、1、N 和 N+1 个轮次。
- 超长单条消息与多语言文本。
- 数字、日期、否定、承诺和纠正移出窗口后的保留。
- 没有记忆字段的旧会话。
- 重复压缩的确定性和有界增长。
- 输入接近模型上下文上限时的降级行为。
- Prompt Cache 前缀未被动态字段意外破坏。
