# 检索、缓存、模型与运行时治理

## RAG 预算

- 记录查询次数、Embedding 输入、候选数、Rerank 数、最终片段数和最终上下文 Token。
- 先执行权限和元数据过滤，再执行高成本 Rerank。
- 分别限制候选、Rerank、最终 Top-K、单片段和总上下文。
- 按稳定片段标识或内容摘要去重。
- 保留引用所需的来源标识和最少原文，不能为了省 Token 破坏证据链。

## Prompt Cache

- 将稳定规则、工具 Schema 和公共资料放在前缀，动态内容放在末尾。
- 规范化字段顺序和序列化方式。
- 使用 Provider 返回的缓存写入、读取或命中 Token 验证效果。
- 同时报告逻辑输入 Token、缓存 Token、实际费用和延迟。

Prompt Cache 通常减少计费或计算，不应被报告为逻辑输入 Token 消失。

## 应用结果缓存

仅缓存答案稳定、时效可控且允许复用的结果。缓存键包含：

- 租户、授权范围，以及必要时的用户和会话。
- 任务、Provider、模型、参数、Prompt 版本和输出 Schema 版本。
- 知识库或索引版本、规范化检索输入和稳定排序的业务输入。

排除非语义时间戳。对敏感字段使用摘要，不在可观察键名中保存原文。定义 TTL、容量、失效、降级和隔离测试。

个性化自由文本、实时数据、权限相关答案和不可验证的语义近似结果默认不缓存。

## Singleflight 与幂等

- 只合并语义输入完全相同且隔离范围一致的并发工作。
- 使用所有者令牌、等待超时、安全过期和所有者失效恢复。
- 为重连、任务、批处理和可重试请求使用稳定幂等键。
- 协调器故障时回退到保守本地并发，不允许无限调用。

## 模型路由与级联

仅在任务级黄金样本和明确授权下考虑：

- 按任务类型、复杂度、上下文长度和能力要求路由。
- 低成本路径通过确定性或独立质量门禁后返回，否则升级。
- 将分类、评分、升级、修复和回退调用全部计入完整流程费用。
- 记录每条路径的质量、回退率、失败率和延迟。
- 保留强模型直达开关和旧路径回滚。

不得根据旧价格表、模型名称或理论能力直接切换。

## 微调与部署选择

- 微调适合高频、稳定、可评测任务；计入训练、数据治理、评测和版本维护。
- 本地或混合部署应计入硬件、能耗、容量、运维、故障、延迟和质量。
- 按总拥有成本和质量比较，不使用“本地推理零成本”的假设。
- 变更模型、Provider 或部署方式前获取单独授权。

## 并发、批处理与流式

- 按 Provider 和模型设置有界并发及有界等待队列。
- 多 Worker 共享配额时使用 Redis 或等效协调机制。
- 独立调用并行执行只降低墙钟时间，不直接减少 Token。
- 批处理仅在多个任务共享指令或上下文时可能减少 Token。
- 合并调用必须验证结果映射、部分失败、上下文窗口和单次失败范围。
- 流式输出改善感知延迟，但不自动减少输出 Token。

## 重试

- 区分限流、超时、临时服务端错误和永久请求错误。
- 使用带抖动的指数退避并尊重 `Retry-After`。
- 限制最大尝试次数、单次超时和完整业务流程总时限。
- 记录重试原因和用量，避免把失败调用排除在费用之外。
- 分布式限流器或队列故障后不得退化为无限重试或无限并发。
