# Token 成本优化

以完整业务结果为单位，通过可测量、可回滚的改动降低模型费用，同时保持质量、安全和公开契约稳定。

## 工作流程

### 1. 确认边界

- 读取仓库指令、架构、调用入口、配置和测试约束。
- 列出受保护契约：Schema、流式事件、错误语义、安全规则、证据、隐私、延迟和质量目标。
- 将模型或 Provider 变更、微调、本地部署、数据库或基础设施变更视为需单独授权的操作。
- 只使用合成数据和环境变量凭据；不得记录 Prompt、完整输出、个人数据或鉴权信息。

### 2. 还原真实调用链

按用户可见的完整业务流程追踪：

- LLM、Embedding、Rerank、语音、图像、工具调用及重试。
- Provider、模型、参数、流式模式、缓存和降级路径。
- 输入、输出、推理、缓存 Token，以及非 Token 计费单位。
- 重复静态指令、完整历史、重复检索、冗长工具定义和并发重复工作。

以运行时代码和用量响应为准，不假设文档、配置名或估算值等于实际行为。

### 3. 建立可复现基线

- 固定合成输入、测试身份、模型版本、时间窗和完整流程。
- 记录任务级调用数、用量来源、Token、费用、延迟、重试、状态和质量结果。
- 分开标记 Provider 返回、Tokenizer 估算和不可用数据。
- 设置最少请求数、完整流程数和失败率门禁；拒绝“零请求也通过”。
- 没有同口径的优化前后基线时，不宣称节省比例。

### 4. 按三层识别热点

按预期节省、质量风险、回滚难度和实施成本排序：

1. **Prompt**：删除重复规则和字段，压缩 Few-shot，使用结构化输出，按历史分位数设置输出上限，稳定可缓存前缀。
2. **Context**：使用滑动窗口与结构化滚动记忆，按需加载工具和资料，为 RAG 候选、Rerank、片段和总上下文设预算。
3. **Harness**：合并重复调用，增加确定性缓存、Singleflight、幂等和有限重试；仅在质量门禁和授权满足后使用模型路由或级联。

并发、批处理和流式主要改善吞吐或感知延迟；只有减少重复调用、重试或重复上下文时才算 Token 优化。

需要选择模式、计算费用或定义测试时，读取 [references/playbook.md](references/playbook.md)。

### 5. 实施单变量改动

- 一次只改变一个优化维度，先做低风险、高证据的热点。
- 保持成功响应、Schema、流式顺序、错误语义和安全行为。
- 为调用编排、上下文压缩、缓存或路由增加回滚开关。
- 为 Prompt、输出 Schema 和知识库设置版本，并纳入相关缓存键。
- 不缓存个性化自由文本，除非产品明确允许且租户、用户、会话隔离已验证。

多轮上下文按以下顺序组装：

```text
稳定规则 + 结构化滚动记忆 + 最近完整轮次 + 当前输入 + 有界检索证据
```

同一内容不得同时出现在记忆、最近轮次、当前输入或检索证据中。

### 6. 验证费用与行为

使用相同合成流程比较优化前后：

- 完整结果及任务级调用数。
- 输入、输出、推理、缓存 Token 和其他计费单位。
- 使用带币种、生效日期和来源的价格计算费用。
- P50/P95 延迟、失败、重试、限流和队列等待。
- Schema、关键事实、引用、安全规则、个性化和流式顺序。

若节省来自删除必要信息、隐性增加修复调用或质量超过容许回归，拒绝该方案。

### 7. 渐进发布

- 从离线回放、影子对比或小流量开始，保留旧路径。
- 监控每个成功业务结果的费用，而非只看单次请求均价。
- 跟踪缓存命中率、缓存 Token、估算用量占比、重试放大、队列饱和和回退率。
- 证据稳定后再扩大范围；触发质量、安全或费用门禁时立即回滚。

## 判断规则

- 不写死厂商价格、模型能力、Tokenizer 比例或节省百分比；以目标环境实测为准。
- Prompt Cache 可能降低计费和延迟，但不会减少发送的逻辑 Token；单独报告原始输入与缓存命中 Token。
- 语义缓存只有在答案稳定、时效可控且隔离可靠时使用。
- 模型路由、级联和微调必须计算升级、训练、评测和维护的总成本。
- 本地部署不是零成本；比较硬件、运维、能耗、容量和质量后的总拥有成本。
- 分布式协调故障后不得退化为无限并发或无限重试。

## 交付内容

- 范围、受保护契约和授权边界。
- 基线口径、数据质量和调用链证据。
- 热点排序、选定方案、被拒方案及原因。
- 修改文件、开关和回滚路径。
- 优化前后的调用数、Token、费用、延迟和质量。
- 未验证项、风险、阻塞项和下一步。
